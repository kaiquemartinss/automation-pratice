# Plano de Teste – Automation Practice

## 1. Objetivo
Garantir a qualidade das principais funcionalidades do site Automation Pratice, validando fluxo de compra, cadastro, login, busca, carrinho e checkout.

---

## 2. Escopo

### ✔ Incluído
- Cadastro
- Login
- Busca de produtos
- Visualização de produtos
- Carrinho
- Checkout
- Contato

### ✖ Fora do Escopo
- Testes automatizados
- Testes de API
- Testes de performance
- Testes mobile nativo
- Testes de segurança
- Testes de carga

---

## 3. Riscos

- Instabilidade do site
- Não há documentação oficial
- Falta de simulação real de pagamento

---

## 4. Critérios de Entrada
- Ambiente disponível
- Acesso ao site
- Mensagens corretas devem aparecer em casos negativos
- Carrinho deve refletir corretamente item, preço, quantidade e total

## 5. Critérios de Saída
- Todos os casos críticos executados
- Bugs críticos corrigidos
- Nenhuma quebra no fluxo principal

---

## 6. Estratégia de Teste
- Testes funcionais
- Testes exploratórios
- Particionamento de equivalência
- Análise de valor limite
- Testes de regressão

---
